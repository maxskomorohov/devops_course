1. Поясніть етапи завантаження системи Linux.

Процесс завантаження Linux:
    1. Boot (з моменту ввімкнення та до запуску systemd)
       Етап від ввімкнення до моменту коли ядро запущено та стартував PID1 (systemd). На цьому етапі система
       переходить від прошивки => до boot loader => до ядра.

       1.1. BIOS/UEFI POST (UEFI - Unified Extensible Firmware Interface)
            Прошивка виконує первинну ініціалізацію платформи: старт CPU, базові налаштування чіпсета та пам'яті,
            діагностика, визначення пристроїв завантаження та передача керування завантажувачу.

       1.2. Boot loader (GRUB2)
            Завантажувач зчитує конфігурацію, може показувати меню вибору, завантажує в RAM ядро та initramfs,
            формує параметри командного рядка ядра та передає керування ядру.

       1.3. Ініціалізація ядра (Kernel initialization)
            Ядро розпаковується та ініціалізує ключові підсистеми (пам'ять, переривання, планувальник, драйвери),
            монтує initramfs, підіймає мінімум, щоб знайти / змонтувати кореневу файлову систему та підготувати
            запуск першого процесу userspace.

       1.4. Запуск systemd
            Після монтування справжнього root, ядро запускає PID1 (/sbin/init, зазвичай це systemd). Це точка переходу
            від ядра до управління системою в userspace.

    2. Startup (після запуску systemd та до підготовки Linux до операційного стану)
       Етап, на якому systemd підіймає userspace по робочого стану: монтування, пристрої, логи, мережа, сервіси,
       логін/GUI - тобто доводить систему до цільового стану (multi-user.target або graphical.target)



2. Як переглянути системні журнали?

Загалом журнали лежать в каталозі /var/log. Журнали є різними як за назвою файла, так і за призначенням:
    * /var/log/syslog - загальні системні повідомлення
    * /var/log/auth.log - — автентифікація, sudo/ssh
    * /var/log/apt/history.log - історія дій apt
    * /var/log/dpkg.log -  історія дій dpkg
    * /var/log/kern.log - повідомлення ядра
    * /var/log/boot.log - повідомлення завантаження
    * journalctl

Файли часто ротуються: *.1, *.gz (стиснені старі логи).

Також для перегляду журналів systemd (журналів конкретних unit-ів) основним інструментом є journalctl:
    journalctl -b  # перегляд логів поточного завантаження
    journalctl -u <unit> # логи конкретного сервісу



3. -rw------- : Опишіть цей дозвіл. Як додати прапорець виконуваного файлу?

Файл з правами на читання (read) та запис (write) для власника (user) та відсутністю будь-яких прав для групи (group)
чи інших (other).

Варіанти зміни execute:
    1. chmod 700 <path>     # перезаписати права, додавши execute до існуючих прав власнику
    2. chmod 710 <path>     # перезаписати права, додавши execute до існуючих прав власнику та групі
    3. chmod ug+x <path>    # додати execute до існуючих прав власнику та групі
    4. chmod a+x <path>     # додати execute до існуючих прав всім (all = user, group, other)


4. Яка різниця між apt і dpkg?

apt і dpkg є пакетними менеджерами для системи сімейства Debian (Ubuntu, Linux Mint) і працюють з deb пакетами, з
тією різницею що:
    * apt є високорівневим менеджером пакетів, а dpkg - низькорівневим;
    * apt встановлює пакети зі всіма необхідними залежностями, перевіряючи також їх версії, конфлікти тощо;
    * при встановленні через apt unit-файли поставляються разом з пакетом (під час встановлення можуть запускатися
    скрипти пакета, які реєструють/вмикають/перезапускають сервіс (через systemctl))
    * apt вміє скачувати з репозиторіїв, а dpkg - ні
